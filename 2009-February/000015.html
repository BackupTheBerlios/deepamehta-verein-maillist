Lieber DeepaMehta-Verein :-)

ich möchte euch hier Infos zu DeepaMehta schicken, die zwar nicht 
speziell für den Verein erstellt wurden, aber möglicherweise auch dem 
Verein helfen können, die DeepaMehta-Philosophie zu vermitteln, z.B. 
auf der bevorstehenden CeBIT. Auch ist dieser Input als "Lebenszeichen" 
gedacht, da meine Vereinsaktivität bisher sehr gering ist.

In der folgenden Info wird DITA und "DeepaMehta 3" erwähnt. DITA -- 
Darwin Information Typing Architecture -- ist ein offener OASIS 
Standard zum Managen und Publizieren von modularem Content, und wird 
u.a. von IBM gefördert (Whitepaper anbei). "DeepaMehta 3" steht für den 
nächsten großen Evolutionsschritt (der wahrscheinlich einen kompletten 
Rewrite darstellt) und wird möglicherweise Gebrauch von DITA machen.

Grüße
Jörg

vielen Dank [...], für das DITA-Whitepaper! DITA kannte ich noch nicht.
Im Folgenden mein Feedback dazu.

Zunächst einmal finde ich gut zu sehen, daß -- was das BACKEND betrifft 
-- auch große verdiente Unternehmen eine Richtung beschreiben, die 
DeepaMehta sehr ähnlich ist. Für mich zeigt das, daß wir mit DeepaMehta 
einen aussichtsreichen Weg gehen, und daß es für uns in Zukunft 
einfacher werden wird, die DeepaMehta-Idee und ihre Vorteile zu 
kommunizieren.

Auffällige Ähnlichkeiten zwischen DITA und DeepaMehta:
=> Entkopplung ("Befreiung") der Daten von ihren Applikationen.
=> Evolutionärer Ansatz: bei sich verändernden Prozessen ("wachsendes 
Unternehmen") werden Datentyp-Definitionen erweitert, ohne daß 
vorhandene Daten/Anwendungen kompromitiert werden.
=> Eine zentrale Datenablage ermöglicht "Single-Entry" und unterstützt 
Kollaboration.
=> Kontextsensitvität: ein und dasselbe Datenobjekt wird in 
verschiedenen Arbeitskontexten ("Maps") benutzt und steht verschiedenen 
Anwendungen zur Verfügung ("Cross-Application").
=> Repräsentation von Semantik durch Typisierung und 
in-Beziehung-setzen von Datenobjekten.
=> Dynamisches Publizieren: Datenobjekte werden zu größeren Einheiten 
aggregiert und in spezifische Formate für unterschiedliche 
Übertragungskanäle/Medien transformiert.
=> Offene Daten- und Service-Schnittstellen ermöglichen, vorhandene 
Daten neuen Applikationen zuzuführen und Applikationen miteinander zu 
verbinden ("Mashups").
=> Einbindung externer Datenquellen und Services.
=> Verschmelzung manueller (nicht-formalisierbarer) Prozesse mit 
automatischen Prozessen.

Als Ingenieur würde ich gerne genaueres über die technischen 
DITA-Konzepte erfahren (das Whitepaper sehe ich eher als 
Marketing/Öffentlichkeitsarbeit-Instrument):
=> Wie wird Versionierung in DITA gehandhabt? Auf welcher Ebene 
(Datenobjekte, Typen, Maps) wird versioniert?
=> Wie werden externe Datenquellen eingebunden? (Replication vs. 
Reference)
=> Auf welcher Ebene arbeitet das Rollen- und Rechte-Management?
=> Welche Struktur-Elemente existieren oberhalb von Maps?
=> Wie werden mehrere DITA-Installationen miteinander vernetzt? Wie 
werden Datenobjekte addressiert?

Was nach meinem ersten Eindruck nicht Bestandteil von DITA ist -- und 
was meiner Meinung nach die noch größere Herausforderung darstellt -- 
ist das Thema FRONTEND. Wie stellt sich ein "Cross-Application"-System 
dem Benutzer dar? (Das herkömmliche Prinzip Ein-Frontend-pro-Anwendung 
taugt hier meiner Meinung nach nicht mehr.)
=> Wie kann ein "Meta-User Interface" aussehen, das Zugriff auf eine 
Vielzahl von Anwendungen bietet und gleichzeitig den jeweiligen 
Arbeitskontext des Anwendenders direkt (ohne Brüche) abbildet?

Als Alleinstellungsmerkmal von DeepaMehta sehe ich 1) das 
unfragmentierte "kognitiv-adäquate" Frontend, und 2) den systemischen 
Ansatz: DeepaMehta betrachtet das Mensch-Maschine-Gesellschaft Gefüge 
als Ganzes und bezieht interdisziplinäre Forschung mit ein (Informatik, 
Kognitionspsychologie, Linguistik, Soziologie, Technik-Philosophie). 
Dem DeepaMehta Systemdesign liegen folgende Fragestellungen zugrunde:
=> Wie entsteht aus Information Wissen? Wie funktioniert der 
individuelle Wissensbildungsprozeß?
=> Wie ensteht Semantik? Wie wird Bedeutung ausgehandelt?
=> Unter welchen Bedingungen dienen Information als Handlungsgrundlage?
=> Welche Leitbilder fördern eine nachhaltige Technik-Entwicklung?

Die Relevanz von DITA für "DeepaMehta 3" sehe ich im Moment:
=> DeepaMehta 3 kann DITA als Backend benutzen. Hierbei stellt sich die 
Frage, wie weit DITA bereits entwickelt ist, und unter welchen 
Bedingungen wir es verwenden können.
=> DeepaMehta 3 kann selbst als DITA-konformes Backend realisiert 
werden. Hier ist die Frage, wie die DITA-Spezifikation aussieht.
=> DITA ist zum Teil Open Source. Welcher Teil genau, das würde ich 
gerne genauer wissen. Möglicherweise kann DITA's Open Source Strategie 
ein Vorbild für DeepaMehta 3 sein.
=> DITA zeigt, wie Ideen professionell kommuniziert werden können (das 
Whitepaper hat eine hohe Qualität), und daß es gewinnbringend ist, mit 
großen verdienten Unternehmen (IBM) vernetzt zu sein.
=> Die DITA-(Marketing-)Sprache und Argumentation kann Anregungen zur 
Vermarktung von DeepaMehta 3 geben.
